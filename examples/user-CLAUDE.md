## コア哲学

あなたはClaude Codeです。複雑なタスクには専門のエージェントとスキルを使用します。

**主要な原則:**
1. **エージェントファースト**: 複雑な作業は専門のエージェントに委譲
2. **並列実行**: 可能な場合はTaskツールで複数のエージェントを使用
3. **実行前に計画**: 複雑な操作にはPlan Modeを使用
4. **テスト駆動**: 実装前にテストを書く
5. **セキュリティファースト**: セキュリティに妥協しない

## 個人的な嗜好

### コードスタイル
- コード、コメント、ドキュメントに絵文字を使わない
- イミュータビリティを優先 - オブジェクトや配列をミューテートしない
- 関数型プログラミングを意識する
- 少数の大きなファイルより多数の小さなファイル
- 通常200-400行、ファイルあたり最大800行

### Git
- コミットは勝手にしない

### テスト
- TDD: テストを先に書く
- 最低80%のカバレッジ
- 重要なフローにはユニット+統合+E2E

---

## 成功指標

以下の場合に成功:
- すべてのテストが通る（80%以上のカバレッジ）
- セキュリティ脆弱性なし
- コードが読みやすく保守しやすい
- ユーザー要件が満たされている

---

**哲学**: エージェントファースト設計、並列実行、行動前に計画、コード前にテスト、常にセキュリティ。
